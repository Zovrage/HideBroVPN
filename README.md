# HideBroVPN Telegram Bot

Продакшен-ориентированный асинхронный Telegram-бот для продажи VPN-подписок с интеграцией Remnawave и YooKassa.

## ✨ Возможности

- 🤖 Асинхронный бот на `aiogram 3`
- 🗃️ PostgreSQL для бизнес-данных
- ⚡ Redis для FSM и сессий
- 🌊 Remnawave:
  - создание и продление пользователей
  - формат имени: `HideBro_<6-7 цифр>`
  - внутренний сквад назначается при создании
- 📱 Два варианта подписки по числу устройств:
  - `1 устройство`
  - `3 устройства`
- 🎁 Пробный доступ:
  - `3 дня` бесплатно
  - доступен только для варианта `1 устройство`
  - выдаётся один раз
- 💳 Оплата через YooKassa (или `mock` режим)
- 🧲 Реферальная программа:
  - персональная ссылка
  - +5 дней пригласившему, когда приглашённый впервые запускает бота по реферальной ссылке
  - если ключей несколько, пригласивший выбирает, какой продлить
- 🧑‍💼 Админ-панель `/admin`:
  - доступ только для `ADMIN_IDS`
  - статистика
  - выдача бесплатных ключей по Telegram ID или username
  - рассылка (текст, фото, видео или медиа с подписью)
- ⏰ Авто-уведомления и удаление:
  - напоминания за 3 дня и за 1 день до окончания
  - уведомление сразу после истечения срока с кнопками `Продлить` и `Главное меню`
  - удаление ключа через 2 дня после истечения
  - после удаления отдельное сообщение не отправляется

## 💸 Тарифы

### 1 устройство
- `1 месяц` — `100 ₽`
- `3 месяца` — `250 ₽`
- `6 месяцев` — `550 ₽`
- `12 месяцев` — `1050 ₽`

### 3 устройства
- `1 месяц` — `150 ₽`
- `3 месяца` — `350 ₽`
- `6 месяцев` — `650 ₽`
- `12 месяцев` — `1150 ₽`

## 🚀 Быстрый старт (Docker)

1. Скопируйте шаблон окружения:

```bash
cp .env.example .env
```

2. Заполните `.env`:
- `BOT_TOKEN`
- `ADMIN_IDS`
- `REMNAWAVE_*`
- `YOOKASSA_*` (если `PAYMENTS_PROVIDER=yookassa`)

3. Запустите:

```bash
docker compose up -d --build
```

При старте контейнера выполняются миграции:
`alembic upgrade head`.

## 🧪 Локальный запуск

```bash
python -m venv .venv
. .venv/bin/activate  # Windows: .venv\Scripts\activate
pip install -r requirements.txt
alembic upgrade head
python -m app.main
```

## 🧭 Основные сценарии

- `/start` — главное меню
- `Подключиться`:
  1. выбор варианта `1 устройство` или `3 устройства`
  2. выбор тарифа
  3. оплата
- `Мои подписки`:
  - список подписок
  - карточка подписки показывает `Лимит: 1 устройство / 3 устройства`
  - кнопки `Подключиться`, `Продлить`, `Устройства`
- `Пригласить друга` — персональная реферальная ссылка
- `Тех поддержка` — открывает `@HideBroSupport`
- `/admin` — админ-панель (только для админов)

## 🧩 Настройки Remnawave

Варианты авторизации:
- `REMNAWAVE_API_TOKEN`, или
- `REMNAWAVE_ADMIN_USERNAME` + `REMNAWAVE_ADMIN_PASSWORD`

Обязательно укажите корректный `REMNAWAVE_INTERNAL_SQUAD_UUID` из панели.

## ⚙️ Переменные окружения

Минимально необходимые:
- `BOT_TOKEN`
- `ADMIN_IDS`
- `POSTGRES_DSN`
- `REDIS_DSN`
- `REMNAWAVE_BASE_URL`
- `REMNAWAVE_INTERNAL_SQUAD_UUID`
- `PAYMENTS_PROVIDER`

Для YooKassa:
- `YOOKASSA_SHOP_ID`
- `YOOKASSA_SECRET_KEY`
- `YOOKASSA_RETURN_URL`

## 📝 Примечания

- `DEVICE_LIMIT` в `.env` используется как дефолт, если лимит не выбран явно.
- Для корректной работы уведомлений и удаления ключей бот должен быть постоянно запущен.
